⚙️ Process

How It Works

Four steps from raw logs to actionable insights.

01
📥
Ingest Logs

Paste or upload raw logs in any format — syslog, JSON, plaintext. The system parses and chunks them automatically.

02
🧮
Embed & Index

Sentence Transformers encode each log chunk into dense vectors stored in a FAISS index for instant semantic search.

03
🔍
RAG Retrieval

Your query retrieves the most relevant log chunks from FAISS, forming a precise context window for the LLM.

04
🤖
AI Analysis

Groq's LLaMA 3 generates a detailed response: root cause, severity classification, and step-by-step fix suggestions.

✨ Capabilities

What LogSense Can Do

Everything you need to debug production incidents faster.

🎯
Root Cause Analysis

Automatically identifies the underlying cause of failures, not just the symptoms, saving hours of manual debugging.

⚠️
Severity Classification

Every finding is tagged with severity: INFO, WARNING, or CRITICAL — so you know what to fix first.

🔧
Fix Suggestions

Get concrete, actionable steps to resolve issues — no more googling stack traces at 3am during incidents.

💬
Conversational Interface

Chat naturally with your logs. Ask follow-up questions, drill into specific services, or get summaries.

⚡
Sub-Second Responses

FAISS vector search + Groq's LPU hardware delivers responses in under a second even on large log files.

🔒
Secure by Design

API keys stored only in .env files, never in code. Logs are processed in-session and not stored externally.

🏗️ About the Project

Built for Real DevOps Problems

LogSense AI is an end-to-end RAG pipeline designed to make log analysis conversational. Instead of grepping through thousands of lines, just ask — and get answers grounded in your actual log data.

The project demonstrates a production-grade architecture: FastAPI backend, sentence-transformer embeddings, FAISS vector indexing, and LLaMA 3 via Groq for blazing-fast inference.
